# Coding standards

Estado del documento: Vigente  
Ultima actualizacion: 2026-05-10

## Reglas

- Componentes pequeños y legibles.
- Separar data de UI.
- Evitar hardcoding innecesario.
- Nombres claros.
- No `console.log` innecesario.
- No colores fuera de paleta.
- No estilos inline salvo justificacion.
- Accesibilidad basica.
- Mobile-first.
- Commits claros.

## React

- Usar `useState`/`useEffect` local mientras sea suficiente.
- Limpiar timers, listeners y sockets.
- No introducir estado global antes de necesitarlo.

## UI

- Mantener estetica dark trading cockpit.
- Evitar morado/lila como branding.
- No prometer resultados financieros.
